## Pre-Engagement
Esta fase es fundamental para obtener la información relevante antes de realizar cualquier evaluación de vulnerabilidades. Es el paso más importante del proceso, ya que reúne datos críticos, como direcciones IP y activos clave, necesarios para establecer una base sólida.
![[Pasted image 20260622120657.png]]
En este apartado, se recomienda tener conocimientos en las siguientes areas:
1. Learning Process
Es fundamental comprender cómo funciona el proceso de aprendizaje humano, ya que esto permite incrementar la eficiencia de manera drástica.

2. Linux Fundamentals
Hoy en día, la mayoría de los servidores utilizan el Kernel de Linux, basándose en distribuciones como Ubuntu, Debian, Arch o RedHat, por lo que su dominio es esencial.

3. Windows Fundamentals
Dado que Windows es el sistema operativo con mayor presencia entre los usuarios finales, resulta indispensable conocer su entorno y funcionamiento.

4. Introduction to Networking
Es esencial comprender cómo se interconecta la información, ya que es la base que permite el funcionamiento de todos los sistemas anteriores.

5. Introduction to Web Applications
La mayoría de los servicios en internet son aplicaciones web. Conocer en qué capa del modelo OSI operan es vital para identificar posibles vectores de ataque.

6. Web Requests
Las aplicaciones web utilizan diversas funciones para procesar información. Comprender su configuración es importante para detectar vulnerabilidades derivadas de una implementación incorrecta.

7. JavaScript Deobfuscation
Las aplicaciones web modernas son dinámicas e interactivas. Entender cómo funciona su código es esencial para identificar y prevenir vulnerabilidades en el lado del cliente.

8. Introduction to Active Directory
La mayoría de las empresas administran cientos o miles de equipos de forma centralizada. Comprender este entorno es clave para detectar posibles riesgos a nivel empresarial.

9. Getting Started
Es necesario desarrollar la capacidad de realizar procesos de forma autónoma, sin necesidad de asistencia, para reforzar y consolidar las habilidades técnicas.
## Information Gathering
Antes de ejecutar cualquier ataque, es indispensable realizar una recolección e identificación de información. Es importante comprender que, por lo general, el cliente solo proporciona datos básicos, como el dominio, rangos de red o direcciones IP, por lo que esta etapa es vital para expandir el conocimiento inicial.
![[Pasted image 20260622120721.png]]
Los pasos posteriores se guiará dependiendo de la obtención de la información, por eso mismo es necesario obtenerla de alguna parte.
1. Network Enumeration with Nmap
	R: Nmap nos brinda distintas herramientas para la detección de intrusos, firewalls, IPs, seguridad, etc. Con estas herramientas podemos obtener mucha más información como las versiones, puertos, etc.
2. Footprinting
	R: Una vez que hayamos identificado el objetivo, necesitamos saber los servicios que brinda el usuario para examinarlos. Es importante saber para que son utilizados esos servicios y saber como atacantes podemos vulnerar sus posibles brechas de seguridad.
3. Information Gathering - Web Edition
	R: Mayormente las paginas contienen información importante y resultan muy vulnerables por la falta de capacitación, por eso mismo debemos enfocarnos en descubrir la estructura de como está construida para aprovechar la vulnerabilidad.
4. OSINT: Corporate Recon
	R: Esta herramienta de busqueda Open Source nos brinda muchisima información que está publica sobre muchos temas, es necesario comprender las limitantes y como nos puede ayudar a conseguir claves SSH, e información relevante.
## Vulnerability Assessment
Este apartado está dividido en dos areas. Uno escanea las vulnerabilidades conocidas utilizando herramientas automaticas y la otra analiza posibles vulnerabilidades con la información encontrada. Realizar analisis es pensar mucho fuera de la caja, ya que debemos ser creativos para encontrar oportunidades para romper sistemas y aplicaciones.
![[Pasted image 20260622124956.png]]
Para este apartado tenemos cuatro caminos que podemos tomar
* Exploitation
* Post-Exploitation
* Lateral Movement
* Information Gathering
Todo dependerá de la profundidad a la que queremos adentrarnos.
Es importante resumir la información que utilizaremos para que las herramientas puedan escanear a los objetivos y ayude a detectar las vulnerabilidades
1. File Transfers
	R: Antes de realizar la explotación, necesitamos estar familiarizados con las tecnicas y metodos de transferencia de archivos para los datos que queremos obtener. Conocer las formas que hay para poder extraer los datos.
2. Shells & Payloads
	R: No solo debemos conocer las maneras de transferir archivos, tambien debemos saber que tipo de archivos transferiremos. 
3. Using the Metasploit-Framework
	R: Existe el framework llamado Metasploit-Framework que cubre muchos de los ataques conocidos, metodos para escalar privilegios y hacer mucho más rapido y eficiente la explotación de la vulnerabilidad.
