# Penetration Testing Process

Un proceso es un término utilizado para describir secuencias directas de eventos. En una organización, el proceso se refiere más precisamente al trabajo, negocio, producción o creación. Aunque puede tener diferentes nombres, este se relaciona directamente con una secuencia de eventos.

Es importante comprender la diferencia entre los procesos deterministas y los estocásticos.

- Un proceso determinista es un proceso en el que cada estado depende causalmente de otro.
- Un proceso estocástico es aquel en el que un estado se sigue de otros estados con una cierta probabilidad.

La definición de *penetration testing process* desde la ciencia social representa un curso de eventos conectados con procesos deterministas, debido a que todos los pasos se basan en eventos y resultados que podemos descubrir y provocar.

> *El proceso de pruebas de penetración es definido por pasos y eventos exitosos realizados por el atacante para encontrar un camino al objetivo definido.*

---

## Pre-Engagement

Es la etapa donde se educa al cliente y se ajusta el contrato. Todas las pruebas necesarias y sus componentes son estrictamente definidos y acordados bajo contrato.

En una reunión cara a cara o llamada de videoconferencia, algunos de los temas que se tratan son:

- Acuerdos
- Metas
- Objetivos
- Tiempo estimado
- Reglas del compromiso

---

## Information Gathering

La obtención de información describe cómo obtenemos información sobre los componentes necesarios de diferentes maneras. Buscamos información del objetivo de la compañía y los posibles espacios de seguridad dentro del software y hardware.

---

## Vulnerability Assessment

Una vez que obtenemos la información del apartado *Information Gathering*, analizamos los resultados obtenidos. Debemos conocer las vulnerabilidades del sistema, aplicaciones y versiones de cada componente para identificar posibles vectores de ataque.

*Vulnerability Assessment* es la evaluación de potenciales vulnerabilidades.

En este apartado se crean los entregables.

---

## Exploitation

En este apartado utilizamos el resultado de nuestro análisis sobre los posibles espacios de vulnerabilidad encontrados, por lo cual se procede a realizar el proceso de explotación para obtener acceso inicial al sistema.

---

## Post-Exploitation

En este apartado ya se debe haber accedido al sistema y se debe asegurar el estado activo, incluso si existen cambios realizados.

Durante esta fase se intenta escalar privilegios para obtener los niveles más altos posibles y conseguir datos sensibles como credenciales u otra información que el cliente quiera proteger.

Muchas veces se realiza post-explotación para demostrar al cliente el impacto del acceso obtenido.

---

## Lateral Movement

Describe el movimiento dentro de la red interna de la compañía objetivo para acceder a dispositivos adicionales con el mismo o mayor nivel de privilegios.

Muchas veces es un proceso iterativo combinado con actividades de post-explotación hasta alcanzar la meta establecida.

Ejemplo: obtener credenciales de una base de datos y encontrar más credenciales para avanzar más profundamente dentro de la red.

Normalmente se utiliza información encontrada en servidores o hosts explotados.

---

## Proof-of-Concept

En este apartado se documenta paso a paso lo realizado, cómo se obtuvo el acceso y qué niveles fueron alcanzados.

El objetivo es explicar cómo fue posible alcanzar la meta planteada, permitiendo que el cliente observe claramente cada vulnerabilidad y pueda priorizar las soluciones.

La documentación detallada ayuda al cliente a identificar fácilmente los posibles problemas.

---

## Post-Engagement

Durante este apartado, la documentación está preparada para los administradores y el cliente de la compañía, con el objetivo de que comprendan la severidad de las vulnerabilidades encontradas.

Se solicita una reunión y, en algunos casos, se realiza una presentación ejecutiva para los directivos.

Posteriormente se genera un archivo donde se almacenan todos los datos y pruebas siguiendo el contrato y las políticas de la compañía.

Finalmente, si el cliente lo requiere, se pueden realizar nuevamente pruebas de penetración siguiendo los mismos pasos.
