

### términos de seguridad

* Activos
Un activo es cualquier cosa con valor para la organización, esto puede ser personar, equipos, recursos, datos, etc.

* Vulnerabilidad
Una vulnerabilidad es una debilidad en un sistema, o su diseño podrá ser explotado por una amenaza.

* Amenaza
Una amenaza es un potencial peligro para los activos, los datos en la red, de una empresa con funcionalidad de enrutamiento.

* Explotar
Una explotación es un mecanismo para tomar ventaja de una vulnerabilidad.

* Mitigación
La mitigación es la contra medida que llega a reducir la probabilidad o gravedad de una posible amenaza o riesgo. la seguirá de la red implica técnicas de mitigación múltiple.

* Riesgo

El riesgo es la probabilidad de que una amenaza explote la vulnerabilidad de un activo, con el objetivo de afectar negativamente a una organización. Riesgo es medido utilizando la probabilidad de ocurrencia de un evento y sus consecuencias.

### Vectores de ataque

Un vector de ataque es una ruta que por la cual una persona que amenazara pueda obtener acceso al servidor, host, red. los vectores se originan dentro o incluso fuera de la red corporativa, las amenazas puedan apuntar a una red través del internet, para interrumpir las operaciones de la red, usualmente con ataques Dos (ataques de denegación de servicio.)

Nota: Un ataque DoS ocurre cuando un dispositivo o aplicación de red está incapacitado y ya no es capaz de admitir solicitudes de usuarios
legítimos.

Un usuario interno, como un empleado o un consultor, puede de manera accidental o intencional:

· Robar y copiar datos confidenciales a dispositivos de almacenaje, correos electrónicos, software de mensajería y otros medios.
· Comprometer servidores internos o dispositivos de infraestructura de red.
. Desconecte una conexión de red crítica y provoque una interrupción de la red.
. Conecte una unidad USB infectada a un sistema informático corporativo.

Las amenazas internas tienen el potencial de causar mayores daños que las amenazas externas porque los usuarios internos tienen acceso
directo al edificio y a sus dispositivos de infraestructura. Los empleados también tienen conocimiento de la red corporativa, sus recursos y sus
datos confidenciales, así como diferentes niveles de usuario o privilegios administrativos.

Los profesionales de seguridad de red deben implementar herramientas y aplicar tecnicas para mitigar las amenazas externas e internas.


### Perdida de datos


Los datos son el activo mas valioso para cualquier organización, lo datos pueden tener muchos orígenes o propósitos, investigación, desarrollo, ventas, finanzas, recursos humanos, asuntos legales, empleados , clientes , etc.



#### vectores de perdida de datos

* correo electrónico/redes sociales.
El correo electrónico o los mensajes de mensajería instantánea interceptado podrían capturarse y descifrar el contenido o información confidencial.

* Dispositivos no encriptados.
si los datos no se almacenan bajo un algoritmo de cifrado, entonces el ladrón podría recuperar datos valiosos o confidenciales.

* Dispositivo de almacenamiento en la nube.
usualmente en estos casos los datos confidenciales se pueden ver comprometidos debido a configuraciones débiles de seguridad débiles.

* Medios extraíbles.
Es un riesgo potencial ya que se conectar USB ajenos ala empresa con malas intenciones y hacer extracciones de datos.
* Respaldo físicos
los datos físicos confidenciales se deben triturar cuando ya no sean necesarios.

* Control de acceso incorrecto.
las contraseñas débiles que se hayan comprometido pueden proporcionar al actor de amenazas con fácil acceso a datos corporativos.

- **Decodificadores de Contraseñas:** Herramientas de recuperación que descifran o recuperan claves eliminando la original, omitiendo el cifrado o por descubrimiento directo (conjeturas repetidas). Ejemplos: John the Ripper, Ophcrack, L0phtCrack, THC Hydra, RainbowCrack y Medusa.

- **Herramientas de Hacking Inalámbrico:** Utilizadas para piratear intencionalmente redes inalámbricas y detectar vulnerabilidades de seguridad. Ejemplos: Aircrack-ng, Kismet, InSSIDer, KisMAC, Firesheep y NetStumbler.

- **Escaneo de redes y Herramientas de Hacking:** Se usan para sondear dispositivos, servidores y hosts en busca de puertos TCP o UDP abiertos. Ejemplos: Nmap, SuperScan, Angry IP Scanner y NetScanTools.

- **Herramientas para Elaborar Paquetes de Prueba:** Sondean y prueban la robustez de cortafuegos (firewalls) mediante el uso de paquetes forjados especialmente diseñados. Ejemplos: Hping, Scapy, Socat, Yersinia, Netcat, Nping y Nemesis.

- **Analizadores de protocolos de paquetes:** Capturan y analizan paquetes dentro de redes tradicionales Ethernet LANs y WLANs. Ejemplos: Wireshark, Tcpdump, Ettercap, Dsniff, EtherApe, Paros, Fiddler, Ratproxy y SSLstrip.

- **Detectores de Rootkits:** Verificadores de integridad de directorios y archivos usados por "sombreros blancos" para detectar grupos de raíz instalados. Ejemplos: AIDE, Netfilter y PF: OpenBSD Packet Filter.

- **Fuzzers para Buscar Vulnerabilidades:** Herramientas usadas por actores de amenazas para descubrir aspectos vulnerables de seguridad en una computadora. Ejemplos: Skipfish, Wapiti y W3af.

- **Herramientas de Informática Forense:** Utilizadas por hackers de sombrero blanco para detectar cualquier rastro de evidencia existente en un sistema. Ejemplos: Sleuth Kit, Helix, Maltego y Encase.

- **Depuradores:** Los hackers de sombrero negro los usan para ingeniería inversa al escribir "debilidades", mientras que los sombreros blancos los usan para analizar malware. Ejemplos: GDB, WinDbg, IDA Pro e Immunity Debugger.

- **Sistemas Operativos para Hacking:** Sistemas especialmente diseñados y precargados con herramientas optimizadas para hackear. Ejemplos: Kali Linux, Knoppix y BackBox Linux.

- **Herramientas de Cifrado:** Utilizan esquemas de algoritmos para codificar datos y evitar el acceso no autorizado. Ejemplos: VeraCrypt, CipherShed, OpenSSH, OpenSSL, Tor, OpenVPN y Stunnel.

- **Herramientas para Atacar Vulnerabilidades:** Identifican si un host remoto es vulnerable a un ataque de seguridad. Ejemplos: Metasploit, Core Impact, Sqlmap, Social Engineer Toolkit y Netsparker.

- **Escáneres de Vulnerabilidades:** Analizan redes o sistemas para identificar puertos abiertos, vulnerabilidades conocidas, máquinas virtuales y dispositivos BYOD. Ejemplos: Nipper, Secunia Nipper, Secuna PSI, Core Impact, Nessus v6, SAINT u Open VAS.
